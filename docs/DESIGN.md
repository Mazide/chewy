# chewy — Design Doc

---

## Персонаж (Hero)

### Концепция
Герой — живое существо, которое отражает качество питания пользователя.
Его состояние меняется в реальном времени: хорошо ешь → герой сильный и счастливый,
игноришь еду или ешь джанк → герой деградирует.

---

### Состояния персонажа

#### Tier 1 — MVP (реализовано)
| Состояние | Триггер | Визуал (сейчас) |
|-----------|---------|-----------------|
| `hungry`  | Старт / после сброса | 🤖 серый, грустный |
| `eating`  | Сразу после Feed | 😋 жёлтый, пульсирует |
| `happy`   | После анализа (2 сек) | 😄 зелёный |

#### Tier 2 — MVP+
| Состояние | Триггер | Идея визуала |
|-----------|---------|--------------|
| `idle`    | Норма, всё ок, просто ждёт | Спокойная idle-анимация |
| `starving` | Не кормили N часов | Вялый, шатается |
| `defeated` | Критически долго без еды | Падает, экран "game over" |

#### Tier 3 — Full RPG
| Состояние | Триггер | Идея визуала |
|-----------|---------|--------------|
| `energized` | Съел много белка | Искры, мускулы |
| `sluggish`  | Джанк / перебор жира | Замедленный, тяжёлый |
| `glowing`   | Идеальный баланс БЖУ | Светится, аура |
| `battle_ready` | Статы в норме 3+ дня подряд | Боевая стойка |
| `leveling_up`  | Достиг нового уровня | Взрыв эффектов |

---

### Маппинг еды на RPG-статы (идея)

```
Белки  → Сила     (ATK)
Углеводы → Энергия  (SPD / Stamina)
Жиры   → Броня    (DEF)
Овощи  → Магия Жизни (HP regen)
```

Баланс всех четырёх = `glowing` / `battle_ready`
Перекос в одну сторону = соответствующий дебафф

## Архитектурные решения

### Спрайт-интеграция
`HeroView.swift` — единственная точка входа для визуала персонажа.
Внутри — `HeroSpriteView` (сейчас заглушка на эмодзи).
При интеграции SpriteKit: заменить только `HeroSpriteView`, интерфейс не трогать.

```swift
// HeroView получает только это:
HeroView(status: appState.heroStatus)

// Внутри переключает анимации по статусу:
switch status {
case .hungry:  // idle animation
case .eating:  // chew animation  
case .happy:   // celebrate animation
}
```

### State Management
`AppState` — единый `ObservableObject`, живёт в `chewyApp`.
Передаётся через `.environmentObject`.
