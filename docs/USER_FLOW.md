## Single Use Flow

Единичное действие — приём пищи.

```
┌─────────────────────────────────────────────────────────────┐
│                        HOME SCREEN                          │
│                   Герой: 😤 HUNGRY                          │
│                                                             │
│              [Лог пуст / прошлые приёмы]                    │
│                                                             │
│                   [ + Add Food ]                            │
└─────────────────────┬───────────────────────────────────────┘
                      │ тап "Add Food"
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                     ADD FOOD SCREEN                         │
│                                                             │
│              [ 📷 Камера / фото еды ]                       │
│                                                             │
│         Portion size  ──●──────────  50%                    │
│                                                             │
│                      [ Feed! ]                              │
└─────────────────────┬───────────────────────────────────────┘
                      │ тап "Feed!"
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                        HOME SCREEN                          │
│                   Герой: 😋 EATING                          │
│              (AI анализирует... 2 сек)                      │
└─────────────────────┬───────────────────────────────────────┘
                      │ через 2 сек
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                        HOME SCREEN                          │
│                   Герой: 😄 HAPPY                           │
│                                                             │
│         🥩 Large portion          13:45                     │
│         [новая запись в логе]                               │
└─────────────────────────────────────────────────────────────┘
                      │ через 3 сек
                      ▼
                  Герой: 😤 HUNGRY
               (готов к следующему приёму)
```

## Day in the Life

Сценарий обычного дня пользователя.
Показывает контекст использования и пробелы в UX.

---

**08:15 — Завтрак**
Открывает приложение после сна.
Герой выглядит вяло — `starving` (не ел с вечера).
Фоткает овсянку с ягодами, ставит порцию 60%.
Герой оживает → `happy`. Лог: "🥣 Medium portion, 08:15".
*Утро начато правильно — герой доволен.*

---

**13:30 — Обед**
Герой снова `hungry` — прошло 5 часов.
На обеде фоткает курицу с гречкой, порция 80%.
AI маппит: белок → Сила ↑, углеводы → Энергия ↑.
Герой → `energized`. В логе две записи.
*Хороший приём — все статы растут.*

---

**16:00 — Перекус (пропущен)**
Герой начинает деградировать — `hungry`.
Пользователь занят, не открывал приложение.
*(Пуш-уведомление? — открытый вопрос)*

---

**19:45 — Ужин**
Открывает приложение — герой `starving`, анимация вялая.
Фоткает пиццу, порция 100%.
AI маппит: много жира → `sluggish`.
Герой наелся, но не `happy` — `sluggish`.
Лог показывает дисбаланс дня.
*Момент для нарратива: "Сегодня перебор с жиром, добавь овощей завтра".*

---

**23:00 — Конец дня**
Герой засыпает / переходит в `idle` ночной режим.
*(Нужен ли ночной режим? — открытый вопрос)*

## Открытые вопросы из флоу

- [ ] Пуш-уведомления — когда и как? ("Герой голодает уже 4 часа")
- [ ] Ночной режим — герой "спит" или просто idle?
- [ ] Что показывать если пользователь не открывал приложение весь день?
- [ ] Нужен ли итог дня — экран "Дневной отчёт"?
- [ ] Как быстро герой переходит hungry → starving → defeated (тайминги)?
